## How to Use

Run this checklist after Manus generates a district research report. Each checkbox verifies a requirement from the prompt. A complete report passes all checks.

---

## Pre-Analysis Setup & Formatting

### Document Header

- [ ] **H1 matches filename:** Document starts with `# [district]_overview_v1.0`
- [ ] **YAML metadata block:** Present with all required fields (district, type, date, prompt, prompt_version, status)
- [ ] **Condensed Header Line:** Single horizontal text block with ratings, PVI, incumbent, map link
- [ ] **Additional Administrative Fields:** Report Type, Last Updated, Next Anticipated Update Trigger

### Version Management

- [ ] **Version number** reflects prompt version (v1.0)
- [ ] **"Last Updated" date** matches the latest entry in the Version History table
- [ ] **Filename** follows convention: `[district]_overview_v1.0.md`

### Consistency Flag

- [ ] If part of a series, confirm structure, terminology, and formatting are consistent with other reports.

---

## Section 0: Administrative & Election Context

- [ ] **Context Table:** All fields are populated with the latest available data.
- [ ] **Primary System Citation:** Statutory citation for primary mechanics is included.

---

## Section I: District Profile and Demographics

### I.A Geographic Overview

- [ ] All components are complete (Counties, Major Population Centers, Economic Character).

### I.B Redistricting Analysis

- [ ] **Redistricting Delta Table** is present and complete.
- [ ] **Redistricting as a System** is documented.
- [ ] **Legal Challenges subsection:** All required fields are present.

### I.C Demographics

- [ ] All data points are included.
- [ ] Data confidence is flagged.

### I.D Economic Profile

- [ ] All components are complete.

### I.E Electoral History

- [ ] **Presidential Performance** (current boundaries) is included for 2024 and 2020.
- [ ] **U.S. House Results** are included for 2024, 2022, and 2020.

---

## Section II: Primary System Context, Institutional Power Dynamics, and Candidate Profiles

### II.A Primary System Context

- [ ] **Primary Type** is specified.
- [ ] **Runoff Threshold** is specified.
- [ ] **Decisive Stage Classification:** Classification is provided with a one-sentence justification.
- [ ] **Unaffiliated Voter Impact:** Assessment is provided with a one-sentence justification.

### II.B Institutional Power Dynamics (REQUIRED)

- [ ] **Minimum Investigation Requirement:** Section contains 3-4 sentences documenting the investigation.
- [ ] **What was investigated:** County party organizations, historical political machines, ballot access rules, endorsement patterns.
- [ ] **Historical context:** Relevant historical context is documented.
- [ ] **Why the conclusion was reached:** Explanation provided for the conclusion.

### II.C Mechanical Risks & Edge Cases

- [ ] All applicable risks are identified.
- [ ] Each risk has a **Relevance Weighting** (High/Medium/Low).
- [ ] Each risk has a **one-sentence justification** for its weighting.
- [ ] Risks are listed in **descending order of relevance**.

### II.D Candidate Profiles

- [ ] **Candidates are correctly classified** as Contenders or Non-Contenders.
- [ ] **Contenders** have full profiles meeting the equal depth requirement.
- [ ] **Strengths & Constraints:** All items are verifiable facts, not subjective observations.
- [ ] **Conditional Qualifications:** Uncertain facts are qualified with conditions (e.g., "as of the report date").

---

## Section III: Campaign Finance

### III.A Candidate Fundraising (2026 Cycle)

- [ ] **Fundraising Table** is complete for all contenders with available data.
- [ ] **Burn Rate:** Included in the table.
- [ ] **Individual vs. PAC Split:** Included in the table.

### III.B Outside Spending (Prior Cycle)

- [ ] Quantitative data is provided (table, absence statement, OR uncertainty flag).
- [ ] **NOT vague:** No statements like "significant spending occurred" without figures.

### III.C Outside Spending (Current Cycle)

- [ ] Status statement is included.

---

## Section IV: Implications by Actor

- [ ] **For Campaigns:** Subsection is present with 3-5 sentences of factual observations.
- [ ] **For Parties & Outside Groups:** Subsection is present with 3-5 sentences of factual observations.
- [ ] **For Voters & Civic Organizations:** Subsection is present with 3-5 sentences of factual observations.
- [ ] **No Strategic Advice:** Section contains only factual observations, not strategic recommendations.

---

## Section V: Open Questions

- [ ] **5-7 Open Questions:** Section contains 5-7 genuinely open factual/structural questions.
- [ ] **Quality Check:** Questions are not rhetorical, not already answered in the report, and not predictions in disguise.
- [ ] **Relevance:** Questions will materially affect the race dynamics as it develops.

---

## Section VI: Data Source Mapping

- [ ] **Data Source Mapping Table** is present and complete.
- [ ] **Outside Spending** is included in the table.
- [ ] **Search Heuristics** section is present.

---

## Section VII: Explicit Exclusions

- [ ] **Verification:** The report contains **NONE** of the prohibited content:
  - [ ] No predictions or forecasts
  - [ ] No strategy recommendations
  - [ ] No strategic advice for candidates
  - [ ] No "path to victory" framing
  - [ ] No normative or moral judgments
  - [ ] No speculation on future outcomes
  - [ ] No broken or hallucinated URLs
  - [ ] No vague quantitative statements
  - [ ] No media interpretation or analysis

---

## Section VIII: Version History

- [ ] **Version History Table** is present and correctly formatted.
- [ ] All versions are recorded.
- [ ] Change descriptions are specific and auditable.

---

## Final Review

### Forensic Standard Compliance

- [ ] **No Speculative Content:** Report contains only forensic facts.
- [ ] **No Strategic Content:** Report contains no strategic advice.
- [ ] **No Media Analysis:** Report does not interpret media coverage.
- [ ] **Strengths & Constraints:** All items are verifiable facts.
- [ ] **URLs Verified:** All URLs in the report are functional.
- [ ] **Quantitative Rigor:** All financial figures are specific, not vague.

### Completeness

- [ ] **Analysis Status:** The "Status" in the YAML header is "Research Complete".
- [ ] **v1.0 Compliance:** All requirements are met.
